# Assignment-005
The goal of this assignment is to develop a comprehensive suite of unit tests for the Calculator program, ensuring that each test is written professionally and is capable of validating the functionality of various methods within the program. By the end of this assignment, your test suite should achieve a minimum of 90% code coverage, reflecting a high degree of thoroughness in testing all critical paths and edge cases.

**Requirements**

**1. Understanding the Codebase:**

Begin by reviewing the Calculator program code to identify all methods, their expected behaviors, input parameters, and any potential edge cases. This should include basic operations (e.g., addition, subtraction, multiplication, division) as well as any additional functionalities (e.g., error handling for division by zero, complex calculations, etc.).

**2. Setting Up the Testing Environment:**

Use a testing framework such as JUnit to write and execute unit tests. Use your development environment-supported code coverage analysis tools to measure your progress toward the 90% goal.

**3. Writing Unit Tests:**

> Construct Individual Test Cases: For each method in the Calculator class, write test cases that cover typical use cases, edge cases, and error cases. Each test should be clear and isolated, focusing on a single method or functionality.

> Use Assertions Judiciously: Validate that method outputs match expected results using assertions (e.g., assertEquals, assertThrows for exceptions).

> Test Edge Cases and Error Handling: Ensure you include cases such as zero, negative numbers, and large inputs. For example, test how the calculator handles division by zero or out-of-range values.

> Continuously run code coverage analysis as you develop tests to ensure you are on track to meet the minimum 90% requirement. Pay particular attention to methods or branches that may be missed in initial testing. Improve your tests as necessary to cover as much of the code as possible.

**4. Documentation and Comments:**

Document each test case with brief comments describing its purpose. For example, "Test division method with positive inputs" or "Ensure exception is thrown when dividing by zero."

**5. Deliverables:**

Submit a well-organized project file containing all unit test files.
At the end of the project README.md: 

> a. Include a summary of your code coverage report that highlights your final coverage percentage.

> b. If any code paths were not covered in testing, provide a short justification.
